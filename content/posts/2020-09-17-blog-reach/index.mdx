### Google Search Console を利用した，検索パフォーマンス監視

このブログでは，[Google Search Console](https://search.google.com/search-console/about?hl=ja) というツールを使って，検索パフォーマンスを追跡できるようにしています。

具体的には，

- **クエリ**：どんな検索キーワードからブログにアクセスしてきてるのか
- **表示回数**：検索結果に表示された回数
- **CTR**：ページごとのクリック率。検索結果でページをみた何割くらいが実際にクリックしているのか
- **掲載順位**：ある検索キーワードに対する，ページごとの表示順位

などがわかるようになっています。

以下の画像は，Google Search Console の画面です。

見にくいですが，記事を増やすのにしたがって「クリック数」と「表示回数」が少しずつ増えているのがわかります。

上の画像はだいたい4ヶ月分くらいの実績ですが，まあ1からブログを立ち上げた割に意外とアクセスがきているなという印象です。

それなりに検索結果に表示されているのは，このブログを生成しているフレームワークである [Gatsby.js](https://www.gatsbyjs.com/) による恩恵（**高速なページのロード**，**簡単に SEO 対策できる**）が大きいのかな〜と思います。

ちなみに，はてなブログでも記事を書いていますが，過去3ヶ月の実績を比較すると以下のようになっています

- **はてな**　　：記事数 18，表示回数 321，クリック数 25
- **このブログ**：記事数 16，表示回数 755，クリック数 37

書いている内容が違うので一概に比較はできませんが，記事数は同等以下なのに，こっちのブログの方が表示回数の点で2倍以上上回っていることがわかります。これは嬉しい事実です。

### 「クエリ」からわかる，ユーザーの関心

[Google Search Console](https://search.google.com/search-console/about?hl=ja) では，ユーザーがどんな検索キーワードを使ってブログにやってくるのか分かるという話をしました。

つまり，ユーザーがどんな関心を持っているのか分かるわけです。

これを使えば，「もともと自分はこういう意図で記事を書いていたけど，○○ というキーワードでたどり着く人もいるんだな〜」という具合に，ユーザーに関する学びが得られます。

実際，このユーザーの暗黙的なフィードバックを利用して [Speakerdeck](/Speakerdeck) に関する記事をちょっと改変してみました。

もともとこの記事は，「[Speakerdeck] のスライドを，React + MDX を使ってこのブログに埋め込んだ方法」というのがもともとのストーリーでした。

ですが，検索クエリを見てみると，Speakerdeck のスライドを，もっと一般に，HTML や Markdown に埋め込むことに関心があるんだな〜というのがわかりました。
React とか MDX とか，このブログ特有の話とかは興味がないわけです。

この洞察から，ブログに以下の変更を加えました。
- ブログ特有の細かい話を後半に移動させて，一般的に役に立つ話を先頭に書く
- 記事のタイトルを，クエリから想定される関心に合致するように修正する

まあこんな具合に，ユーザーのアクセスを増やすことや，満足度を向上させるための施策を考えるのに役に立つわけです。

### これからやりたいこと

Google Search Console を活用して，ほかにもこんなことをやりたいな〜と思っています。

- 記事の見せ方（タイトル，サムネイル画像）とアクセス数について検証するための A/B テスト
- ユーザーの関心に合わせて記事を改変すると，どの程度検索パフォーマンスが改善するか
- ページの読み込みに関する改善が，どの程度検索パフォーマンスが改善するか
